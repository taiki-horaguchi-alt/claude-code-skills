# X/Twitter Growth Engine

Source: [alirezarezvani/claude-skills](https://github.com/alirezarezvani/claude-skills) (MIT, 7k stars)

既存のx-daily, x-trend, x-writingと補完関係。このスキルはX特化の成長戦略に深く入る。

## Step 1 - Profile Audit

### Bio Checklist
- [ ] 1行目に明確なバリュープロポジション（誰を + どう助けるか）
- [ ] 具体的なニッチ（「起業家 | 思想家」はNG）
- [ ] ソーシャルプルーフ（フォロワー数、肩書き、実績）
- [ ] CTA or リンク（ニュースレター、プロダクト）
- [ ] ハッシュタグなし（アマチュア感が出る）

### Pinned Tweet
- [ ] 30日以内のもの
- [ ] ベストワークか最強のフック
- [ ] 明確なCTA

### Recent Activity (直近30投稿)
- [ ] 投稿頻度: 最低1日1回、理想3-5回
- [ ] フォーマットMIX: ツイート、スレッド、リプライ、引用
- [ ] リプライ比率: 全活動の30%以上

## Step 2 - Competitive Intelligence

1. `site:x.com "トピック" min_faves:100` で高パフォーマンス投稿を検索
2. ニッチ内の5-10アカウントを特定
3. 分析: 投稿頻度、コンテンツタイプ、フックパターン、エンゲージメント率

### Extract
- **フックパターン**: トップ投稿の書き出し（質問？大胆な主張？統計？）
- **コンテンツテーマ**: 最もエンゲージメントが高い3-5トピック
- **フォーマットMIX**: ツイート vs スレッド vs リプライ vs 引用の比率
- **投稿時間**: ベスト投稿のタイミング

## Step 3 - Content Creation

### 1. Threads (最高リーチ、最高フォロー転換)
```
- Tweet 1: フック（7語以内でスクロールを止める）
- Tweet 2: コンテキストか約束（「学んだことはこれ:」）
- Tweets 3-N: 1ツイート1アイデア、単独でも成立
- Final: まとめ + 明確なCTA
- Reply to tweet 1: フック再提示 + 「フォローで[トピック]をもっと」

ルール:
- 5-12ツイートが最適
- 各ツイートは単独で意味が通る
- 改行で読みやすく（3-4行MAX）
```

### 2. Atomic Tweets
```
有効フォーマット:
- 観察: 「[X]は過小評価されている。理由:」
- リスト: 「毎日使う10ツール:\n\n1. X - Y用」
- 逆張り: 「Unpopular opinion: [主張]」
- 教訓: 「[X]を[期間]やった。最大の学び:」
- フレームワーク: 「[概念]を30秒で説明:」

ルール:
- 200文字以下がエンゲージメント高い
- リンクは本文に入れない（リーチ激減）→ リプライに置く
```

### 3. Quote Tweets (権威構築)
- オリジナルが見落としたデータを追加
- 反論やニュアンスを提供
- 「これ」「ほんとそれ」だけはNG

### 4. Replies (最速の可視性獲得)
- 自分の2-10倍のアカウントにリプライ
- 「素晴らしい投稿！」ではなく価値を追加
- 大きなアカウントに最初にリプライ

## Step 4 - Algorithm Mechanics (2025-2026)

| Signal | Weight | Action |
|--------|--------|--------|
| リプライ受信 | Very High | リプライを誘発するコンテンツ（質問、議論） |
| 読了時間 | High | スレッド、改行のある長めのツイート |
| プロフィール訪問 | High | 好奇心ギャップ、専門性を匂わせる |
| ブックマーク | High | 保存したくなるコンテンツ（リスト、フレームワーク） |
| RT/引用 | Medium | シェアしたくなるインサイト |
| いいね | Low-Med | 共感しやすいコンテンツ |
| リンククリック | Low (penalized) | **絶対に本文にリンクを入れない** |

### リーチを殺すもの
- 本文にリンク（リプライに入れる）
- 投稿後30分以内の編集
- 投稿直後にオフライン
- ハッシュタグ3つ以上
- 品質が不均一なスレッド

### 最適投稿頻度

| アカウント規模 | ツイート/日 | スレッド/週 | リプライ/日 |
|--------------|-----------|-----------|-----------|
| < 1K | 2-3 | 1-2 | 10-20 |
| 1K-10K | 3-5 | 2-3 | 5-15 |
| 10K-50K | 3-7 | 2-4 | 5-10 |
| 50K+ | 2-5 | 1-3 | 5-10 |

## Step 5 - Growth Playbook

### Week 1-2: Foundation
1. Bio + ピン留めツイート最適化
2. 毎日エンゲージするアカウント20個特定
3. 1日10-20リプライ（価値提供のみ）
4. 1日2-3ツイートでフォーマットテスト
5. 週1スレッド公開

### Week 3-4: Pattern Recognition
1. 最もエンゲージメント高いフォーマットを特定
2. トップ2フォーマットに集中
3. 1日3-5投稿に増加
4. 週2-3スレッド
5. 毎日引用ツイート

### Month 2+: Scale
1. 定期シリーズ3-5本（「金曜フレームワーク」等）
2. クロスポスト: スレッド → LinkedIn, ニュースレター
3. 同規模アカウント5-10個と相互エンゲージメント
4. スペース/音声の実験

## Common Pitfalls

1. **本文にリンク** → 必ずリプライに
2. **スレッドTweet 1が弱い** → フックが全て
3. **投稿が不規則** → アルゴリズムは毎日の一貫性を報酬
4. **発信だけ** → 成長の50%以上はリプライとエンゲージメント
5. **汎用的なBio** → 具体性がないと誰にも刺さらない
